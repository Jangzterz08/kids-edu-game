## MiniGame gameBadge Label Contract

This is the only UI-visible change in Phase 8. The `gameBadge` ternary in `client/src/pages/MiniGame.jsx` (lines 66–73) currently falls through to `'❓ Quiz'` for any unrecognized `gameType`. Three new labels must be added to prevent visual confusion when Phase 9 game types are tested end-to-end in Phase 8 integration tests.

| gameType string | Badge label | Emoji |
|-----------------|-------------|-------|
| `'sort'` | `'🔢 Sort'` | Numeric emoji — matches ordering/sorting concept |
| `'trueFalse'` | `'✅ True or False'` | Checkmark — matches binary choice concept |
| `'memoryMatch'` | `'🧠 Memory Match'` | Brain — matches recall/memory concept |

**Implementation:** Add three entries to the existing ternary chain at `client/src/pages/MiniGame.jsx` lines 66–73. Follow the exact same pattern as the existing entries. No new component, no new file.

**Fallback behavior:** After these entries are added, any unrecognized `gameType` continues to fall through to `'❓ Quiz'` — the existing final branch is preserved unchanged.

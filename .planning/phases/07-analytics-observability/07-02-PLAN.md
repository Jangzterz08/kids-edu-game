**Step 2 — client/src/main.jsx:**
Add `import * as Sentry from '@sentry/react';` as the FIRST import line (before react-dom).
After all imports, before `createRoot`, add:
```javascript
Sentry.init({
  dsn: import.meta.env.VITE_SENTRY_DSN,
  environment: import.meta.env.MODE,
  enabled: import.meta.env.MODE !== 'development',
  tracesSampleRate: 0.1,
  integrations: [Sentry.browserTracingIntegration()],
});
```
In the existing `<ErrorBoundary>` `onError` prop, add `Sentry.captureException(error, { extra: info });` after the existing `console.error` call. This captures errors via the `onError` callback prop -- intentionally NOT wrapping with `<Sentry.ErrorBoundary>` to avoid double boundary nesting. The `onError` callback approach is simpler and achieves the same result (Sentry receives the error with stack trace) without introducing a redundant nested error boundary. Do NOT replace or wrap `<ErrorBoundary>` with `<Sentry.ErrorBoundary>`.

**Step 3 — client/vite.config.js:**
Add import: `import { sentryVitePlugin } from '@sentry/vite-plugin'`
Add `build: { sourcemap: true }` to the config object.
Add `sentryVitePlugin({ org: process.env.SENTRY_ORG, project: process.env.SENTRY_PROJECT, authToken: process.env.SENTRY_AUTH_TOKEN })` to the plugins array AFTER VitePWA.

**Step 4 — client/vercel.json:**
In the CSP header value, change:
```
connect-src 'self' https://*.supabase.co https://kids-edu-game-production.up.railway.app
```
to:
```
connect-src 'self' https://*.supabase.co https://kids-edu-game-production.up.railway.app https://*.ingest.sentry.io
```

**Step 5 — server/src/index.js:**
Move `require('dotenv').config();` to line 1 (so SENTRY_DSN is available from .env).
Add as lines 2-9:
```javascript
const Sentry = require('@sentry/node');
Sentry.init({
  dsn: process.env.SENTRY_DSN,
  environment: process.env.NODE_ENV || 'development',
  enabled: process.env.NODE_ENV === 'production',
  tracesSampleRate: 0.1,
  integrations: [Sentry.expressIntegration()],
});
```
Before `app.listen(...)`, add:
```javascript
process.on('unhandledRejection', (reason) => {
  Sentry.captureException(reason);
});
```
Before the existing error handler middleware `app.use((err, req, res, next) => { ... })`, add:
```javascript
app.use(Sentry.expressErrorHandler());
```

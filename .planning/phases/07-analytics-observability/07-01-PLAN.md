  <action>
For each of the 8 game components, apply the following pattern:

1. **Root wrapper:** The outermost `<div>` of each game component must have style `{ display: 'flex', flexDirection: 'column', height: '100%', overflow: 'hidden' }`. Remove any fixed `padding` using `var(--space-xl)` or large pixel values — use `padding: '8px 12px'` max.

2. **Question/prompt area:** Use `flex: '0 0 auto'` so it takes only the space it needs (image, question text, progress dots).

3. **Answer/choice area:** Use `flex: '1 1 0'` with `overflowY: 'auto'` and `minHeight: 0` so it fills remaining space and scrolls internally if choices overflow.

4. **Remove any fixed pixel heights** on game cards, option buttons, or question containers that would cause overflow. Use `maxHeight` with percentage or viewport-relative values instead.

5. **Image sizing:** Any question images should use `maxHeight: '25vh'` (not fixed pixel heights like 200px) with `objectFit: 'contain'`.

Specific per-component notes:
- **QuizGame.jsx:** The image + word area is the prompt (flex: 0 0 auto). The 4 answer buttons are the choice area (flex: 1 1 0, overflowY: auto).
- **MatchingGame.jsx:** The grid of cards is the main area — use `flex: 1 1 0` with `overflowY: auto` on the grid container.
- **SpellingGame.jsx / WordScramble.jsx:** Letter tiles area should be `flex: 1 1 0` with internal scroll.
- **TracingGame.jsx:** The canvas/tracing area is the main content — use `flex: 1 1 0` with no scroll (canvas fills available space).
- **PatternGame.jsx:** Pattern display + choices layout — pattern display `flex: 0 0 auto`, choices `flex: 1 1 0`.
- **PhonicsGame.jsx:** Audio prompt area `flex: 0 0 auto`, answer choices `flex: 1 1 0`.
- **OddOneOutGame.jsx:** Grid of items — use `flex: 1 1 0` with `overflowY: auto`.

Do NOT change any game logic, scoring, or callbacks — only layout styles.

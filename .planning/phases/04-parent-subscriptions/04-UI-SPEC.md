## Spacing Scale

Declared values (all multiples of 4, drawn from `--space-*` tokens in `client/src/index.css:96–101`):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, lock icon internal offset (`top: 8px; right: 8px` — see locked ModuleCard) |
| sm | 8px | Compact element spacing, plan picker card internal padding |
| md | 16px | Default element spacing, subscription section card padding |
| lg | 24px | Section padding, plan picker gap between cards |
| xl | 40px | Layout gaps (existing token value — preserved for backward compatibility) |
| 2xl | 64px | Page-level spacing |

**Source:** `--space-xs` through `--space-2xl` in `client/src/index.css`.

Exceptions:
- Touch targets (upgrade button, manage billing button): minimum 44px height — WCAG 2.5.5 compliance.
- `--btn-height: 72px` — existing primary CTA height; used for "Upgrade" and "Start Learning" buttons.
- Plan picker cards: 160px minimum height, side-by-side layout with `gap: 16px`.
- Locked ModuleCard lock icon: `position: absolute; top: 8px; right: 8px; font-size: 20px` — 8px matches `--space-sm`.

## Color

Ocean palette — all tokens already defined in `client/src/index.css:11–101`.

| Role | Value | Token | Usage |
|------|-------|-------|-------|
| Dominant (60%) | `#3BBFE8` | `--bg-primary` | Full-page ocean gradient background (unchanged) |
| Secondary (30%) | `rgba(255,255,255,0.38)` | `--glass-bg` | Subscription section card, plan picker cards, upgrade banner |
| Accent (10%) | `#10B981` | `--green` | "Best value" badge on annual plan card, active subscription status pill |
| Destructive | `#EF4444` | `--accent-red` | `past_due` subscription status indicator, payment failed state text |

**Accent reserved for:** "Best value" badge background on the annual plan picker card, and the active/trialing subscription status pill border. No other new UI in Phase 4 uses `--green` as accent.

**Locked ModuleCard overlay:** Uses `opacity: 0.45` on the entire card (established in CONTEXT.md specifics: "~40% opacity"). The lock icon itself is `🔒` rendered at `font-size: 20px` — no color override, emoji color is system-default.

**Upgrade banner:** Uses `--glass-bg` with a `--primary` (`#0EA5E9`) left-border accent (`border-left: 4px solid var(--primary)`) to draw attention without using a loud background color. This is a parent-facing surface — adult tone, not the high-saturation kid game palette.

**Plan picker — selected state:** Selected card gets `border: 2px solid var(--primary)` and `background: rgba(14,165,233,0.12)` to indicate selection without an accent-heavy fill.

## Interaction States

### Locked ModuleCard (MON-01, MON-05)

| State | Visual |
|-------|--------|
| Free module (alphabet, numbers, shapes) | Normal opacity (1.0), fully interactive — unchanged |
| Locked module, isPremium=false | `opacity: 0.45`, `pointer-events: none`, `🔒` icon at `position: absolute; top: 8px; right: 8px; font-size: 20px` |
| Locked module, isPremium=true | Normal opacity (1.0), fully interactive — no lock shown |

Kids do not see a paywall screen or dialog. The lock is purely visual — no tap/click action fires.

**Source:** CONTEXT.md — "Locked ModuleCards: rendered with 🔒 icon overlay and reduced opacity; `pointer-events: none`" + CONTEXT.md specifics: "🔒 icon + ~40% opacity on the ModuleCard."

### Upgrade Banner (MON-05)

Shown in `ParentDashboard` at top of page, above kid selector tabs, when `subscriptionStatus` is NOT `active`.

| State | Visual |
|-------|--------|
| `trialing` | Glass card with `border-left: 4px solid var(--primary)`. Heading: "X days left in your free trial". Subtext: "Upgrade to unlock all 13 modules after your trial ends." CTA button: "Upgrade Now" (`.kid-btn` blue style, 44px+ height). |
| `none` or `canceled` | Glass card. Heading: "Unlock all 13 modules." Subtext: "Your kids are on the free plan — only 3 modules available." CTA button: "Upgrade Now". |
| `past_due` | Glass card with `border-left: 4px solid var(--accent-red)`. Heading: "Payment issue — modules locked." Subtext: "Update your payment method to restore access." CTA button: "Fix billing" (opens Stripe Customer Portal). |
| `active` | Banner not rendered. |

### Plan Picker (MON-02)

Shown inline inside `SubscriptionSection` when parent clicks "Upgrade Now" (replaces the banner, or shows below it as an expanded section).

| Element | Visual |
|---------|--------|
| Monthly card | Glass panel, `--font-md` price display "$4.99/mo", `--font-sm` label "Billed monthly", no badge |
| Annual card | Glass panel, `--font-md` price display "$39.99/yr", `--font-sm` label "Billed annually", green "Best value" badge (top-right), `--font-xs` secondary "Save 33%" |
| Unselected card | `border: 1.5px solid var(--glass-border)` |
| Selected card | `border: 2px solid var(--primary)`, `background: rgba(14,165,233,0.12)` |
| Confirm CTA | `.kid-btn` blue style, full-width, label: "Start Free Trial" (if still in trial window) or "Subscribe" (post-trial). Disabled until a plan card is selected. |
| Cancel/back | Ghost text link: "Maybe later" below the CTA — no button styling, `--text-secondary` color. |

### Subscription Management Section (MON-04)

Shown in `SubscriptionSection` when `subscriptionStatus === 'active'`.

| Element | Visual |
|---------|--------|
| Plan label | Fredoka 600 22px: "Monthly Plan" or "Annual Plan" |
| Status pill | Rounded pill, color per status table above |
| Next billing date | Fredoka 400 16px: "Next billing: March 28, 2026" |
| Price reminder | Fredoka 400 16px: "$4.99 / month" or "$39.99 / year" |
| Manage billing CTA | Ghost button style (`rgba(255,255,255,0.2)` bg, white border, white text), label: "Manage billing" — opens Stripe Customer Portal in new tab |

### Stripe Checkout Flow (MON-02)

| Event | What happens |
|-------|-------------|
| Parent clicks "Subscribe" / "Start Free Trial" | Client POSTs to `/api/billing/checkout`, receives `{ url }`, executes `window.location.href = url` — full redirect to Stripe-hosted Checkout page |
| Checkout success | Stripe redirects to `/parent?checkout=success` — show `toast.success("Subscription activated! All modules unlocked.")` on mount if query param present |
| Checkout cancel | Stripe redirects to `/parent?checkout=cancel` — show `toast("Checkout cancelled — you can upgrade any time.")` on mount if query param present |
| Checkout session creation fails | `toast.error("Couldn't start checkout — please try again.")` — do not redirect |

### Loading States

| Action | Loading indicator |
|--------|------------------|
| Fetching subscription status (home-summary) | Subscription section renders skeleton: gray rounded rect at same height as card, pulsing opacity animation (existing `pulse-glow` keyframe) |
| "Upgrade Now" button clicked (waiting for checkout URL) | Button text changes to "Opening Stripe…", disabled, no spinner needed |
| "Manage billing" clicked (waiting for portal URL) | Button text changes to "Opening portal…", disabled |

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA — upgrade (trial active) | "Upgrade Now" |
| Primary CTA — upgrade (no subscription) | "Upgrade Now" |
| Primary CTA — checkout confirm (trial window open) | "Start Free Trial" |
| Primary CTA — checkout confirm (post-trial) | "Subscribe" |
| Primary CTA — billing portal | "Manage billing" |
| Plan picker cancel | "Maybe later" |
| Upgrade banner heading — trialing | "X days left in your free trial" |
| Upgrade banner body — trialing | "Upgrade to unlock all 13 modules after your trial ends." |
| Upgrade banner heading — no plan | "Unlock all 13 modules." |
| Upgrade banner body — no plan | "Your kids are on the free plan — only 3 modules available." |
| Upgrade banner heading — past_due | "Payment issue — modules locked." |
| Upgrade banner body — past_due | "Update your payment method to restore access." |
| Checkout success toast | "Subscription activated! All modules unlocked." |
| Checkout cancel toast | "Checkout cancelled — you can upgrade any time." |
| Checkout error toast | "Couldn't start checkout — please try again." |
| Portal error toast | "Couldn't open billing portal — please try again." |
| Empty state — no kids added | Not applicable — subscription section is hidden when no kids exist; parent is prompted to add a kid first (existing behavior unchanged) |
| Subscription status — trialing | "Free Trial" |
| Subscription status — active | "Active" |
| Subscription status — canceled | "Cancelled" |
| Subscription status — past_due | "Payment issue" |
| Subscription status — none | "Free plan" |
| Monthly plan label | "Monthly Plan" |
| Annual plan label | "Annual Plan" |
| Annual "Best value" badge | "Best value" |
| Annual save label | "Save 33%" |
| Monthly price display | "$4.99 / month" |
| Annual price display | "$39.99 / year" |
| Next billing label | "Next billing: {date}" |
| Trial end label | "Trial ends: {date}" |
| Destructive — cancel subscription | Not in-app. Handled by Stripe Customer Portal. No confirmation dialog needed in this phase. |

**Tone:** Warm, low-pressure. Trial copy creates gentle urgency ("X days left") without alarm. Payment issue copy is factual, not alarming — "Payment issue" not "Payment failed". Avoid "expired", "blocked", "denied". Prefer "unlock" over "buy". Copy targets parents (adult register) not kids.

**Trial countdown calculation:** "X days left" uses `Math.ceil((trialEndsAt - Date.now()) / 86400000)`. Display "1 day left" not "0 days left" — when less than 24 hours remain, round up.

## Accessibility Notes

- Locked `ModuleCard`: `aria-disabled="true"` and `aria-label="{Module name} — requires premium subscription"` on the locked card wrapper. This communicates the locked state to screen readers even though `pointer-events: none` prevents mouse interaction.
- Upgrade banner: `role="status"` — informational, not urgent. `past_due` state uses `role="alert"` — needs immediate screen reader announcement.
- Plan picker cards: `role="radio"` on each card, `aria-checked` toggled on selection, wrapped in `role="radiogroup"` with `aria-label="Choose your plan"`.
- "Manage billing" button: `aria-label="Manage billing — opens Stripe Customer Portal in a new tab"`.
- All new buttons meet 44px minimum touch target height.
- Toast notifications inherit Phase 2 sonner configuration (richColors, top-center position, prefers-reduced-motion handled natively by sonner).

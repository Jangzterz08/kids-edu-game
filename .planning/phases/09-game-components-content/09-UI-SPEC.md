## Spacing Scale

Declared values (all multiples of 4, sourced from `--space-*` tokens in `index.css`):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`--space-xs`) | Icon gaps, badge padding |
| sm | 8px (`--space-sm`) | Compact element spacing, inner card padding |
| md | 16px (`--space-md`) | Default element spacing, gap between game items |
| lg | 24px (`--space-lg`) | Section padding, container insets |
| xl | 40px (`--space-xl`) | Layout gaps, button horizontal padding — pre-existing `--space-xl` token from `client/src/index.css`, not introduced by Phase 9 |
| 2xl | 64px (`--space-2xl`) | Major section breaks, page-level spacing |

Exceptions:
- Touch targets (True/False buttons, Sort items, MemoryMatch cards) — minimum tap area 44px height, achieved via `min-height: 44px` or the global `--btn-height: 72px` for primary action buttons
- MemoryMatch card grid: 4×3 layout uses explicit card width/height values sized to fit within the game viewport; recommended 72px × 72px per card on mobile (16px gap between cards)
- Sort `renderSize` values (24px, 48px, 80px for ages 4–6 items) — these are visual rendering sizes for emoji items, NOT spacing tokens; they are part of lesson data, not the design system

**Source:** `index.css` lines 95–101, `--btn-height` line 81

## Typography

All type uses Fredoka. Four sizes declared from the existing `--font-*` token scale.

| Role | Size | Weight | Line Height | Token |
|------|------|--------|-------------|-------|
| Body | 18px | 400 (regular) | 1.5 | `--font-base` |
| Label | 16px | 400 (regular) | 1.4 | `--font-sm` |
| Heading | 22px | 600 (semibold) | 1.3 | `--font-md` |
| Display | 28px | 600 (semibold) | 1.2 | `--font-lg` |

**Source:** `index.css` lines 86–93, 109

**Weights:** exactly 2 — 400 (regular) and 600 (semibold). No other weights are used in Phase 9.

Rules for this phase:
- TrueFalse claim text (ages 6–8): 22px / 600 (Heading role)
- TrueFalse emoji display (ages 4–6): rendered as text emoji, 48px effective size via `fontSize: '48px'`; no weight declaration needed
- Sort prompt text: 22px / 600 (Heading role) — "Smallest to biggest!"
- Sort item emoji: sized by `renderSize` lesson data field, not font-size token
- MemoryMatch: no text labels — emoji only on card faces
- True/False button labels: 22px / 600 (semibold) — use 600 everywhere including `.kid-btn`; do NOT override with `font-weight: 700`

## Component-Specific Visual Contracts

### TrueFalseGame

**Layout:** Two large buttons stacked vertically (not side-by-side) — better for small screens and larger tap targets. Full-width buttons inside a centered container (max-width 360px on desktop, full-width on mobile).

**Claim display:**
- Ages 4–6: Single emoji centered at `fontSize: '64px'`, no text. Replay button (🔊) below emoji at `--font-md` (22px).
- Ages 6–8: Text claim at Heading size (22px / 600), spoken on mount. Replay button (🔊) inline at end of text or below.

**Button states:**
- Neutral: `.kid-btn` blue (`--btn-blue-base`) — full-width, `--btn-height` (72px), `--btn-radius` (36px)
- Correct: background transitions to `--btn-green-base`, checkmark appended to label (✔), `pop-correct` animation (0.55s)
- Wrong: background transitions to `--accent-red`, animation `shake 0.5s ease`. Correct button simultaneously highlights green.
- Disabled after answer: both buttons `pointer-events: none` until auto-advance

**Answer labels:** "True ✓" / "False ✗" in neutral state. Labels update on selection: correct answer shows ✔, wrong answer shows ✖.

### MemoryMatchGame

**Grid:** 4 columns × 3 rows, 12 cards total (6 pairs). CSS Grid: `grid-template-columns: repeat(4, 1fr)`, `gap: 8px` (`--space-sm`). Container max-width: 320px centered.

**Card dimensions:** 72px × 72px. `border-radius: var(--card-radius)` (24px).

**Card back design:** "?" character at `fontSize: '28px'`, `fontWeight: 600`, `color: '#FFFFFF'` on `--btn-blue-base` background. (Recommended over 🧠 brain — matches MatchingGame's question-mark back convention.)

**Card flip:** CSS 3D transform. Container has `perspective: 600px`. Inner element uses `transformStyle: 'preserve-3d'`, `transition: 'transform 0.4s ease'`. Both `backfaceVisibility: 'hidden'` and `WebkitBackfaceVisibility: 'hidden'` on front and back faces. Flipped state: `transform: 'rotateY(180deg)'` on inner element.

**Card states:**
- Unflipped: Blue back (`--btn-blue-base`), "?" visible, `boxShadow: '0 8px 0 var(--btn-blue-shade)'`
- Flipped (evaluating): White glass surface (`--glass-bg`), emoji visible at `fontSize: '36px'`
- Matched: `--btn-green-base` background, `animation: 'match-bounce 0.55s ease'`, `boxShadow: '0 0 24px rgba(16,185,129,0.5)'`, `pointer-events: 'none'`
- Locked (mismatch evaluation window): no visual change — cards flip back after 900ms via `setTimeout`

### SortGame

**Layout:** Items displayed in a row (horizontal flex), evenly spaced. Container uses `touchAction: 'none'` to prevent scroll hijacking on iOS.

**Item rendering:** Each item is a button with emoji rendered at `fontSize: item.renderSize + 'px'`. Minimum item button width: 64px. Items centered within their button. Label text (ages 6–8 only) shown below emoji at `--font-xs` (14px).

**Item states:**
- Neutral: `--glass-bg` surface, `--glass-border` border, `--card-radius` radius, `box-shadow: var(--shadow-sm)`
- Selected: Cyan glow border (`--accent-cyan`), `boxShadow: '0 0 0 3px var(--accent-cyan)'`, scale `1.05` transform — distinguishes selected from unselected
- Correct (post-Check): `--btn-green-base` background, no animation
- Wrong (post-Check): `--accent-red` background, `animation: 'shake 0.5s ease'`

**Check button:** Centered below the sort row. Uses `.kid-btn` class with green variant. Label: "Check ✓". Positioned with `marginTop: 'var(--space-lg)'` (24px). Button only enabled after all items have been placed (if using a slot-based approach) OR always enabled (tap-to-swap approach — always enabled since items start in a shuffled order and kid rearranges them).

**Results reveal:** Correct items glow green, wrong items shake red simultaneously on Check tap. Score displayed as fraction (e.g., "3 / 4 correct!") at Heading size for 1.5s before `onComplete(score)` is called.

## Interaction Contract

### Touch / Pointer

- All interactive elements are `<button>` elements — `onClick` fires on both mouse and touch on all devices
- SortGame container: `touchAction: 'none'` to prevent page scroll during item placement
- MemoryMatch locked state: `pointerEvents: 'none'` on all cards during the 900ms mismatch evaluation window (prevents 3-card-flipped bug)
- Global: `touch-action: manipulation` on all `button` elements from `index.css` line 7–9 removes 300ms tap delay

### Auto-Advance Timing

- After `onComplete(score)` is called with a 700ms delay (matching existing pattern from MatchingGame)
- MiniGame.jsx handles the sequence — game components do not navigate
- Brief results display duration: ~1.5s visual (managed by the `setTimeout` in the game component before calling `onComplete`)

### Locked State (MemoryMatch)

- `locked` boolean state prevents new card taps during 900ms mismatch flip-back window
- Pattern sourced from MatchingGame.jsx line 37

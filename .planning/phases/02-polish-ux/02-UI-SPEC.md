## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding between label and icon |
| sm | 8px | Compact element spacing, offline banner internal padding |
| md | 16px | Default element spacing, toast internal padding |
| lg | 24px | Section padding, card inner padding |
| xl | 40px | Layout gaps between major UI sections |
| 2xl | 64px | Page-level spacing |

**Source:** `--space-xs` through `--space-2xl` in `client/src/index.css:96–101`.

Exceptions:
- Touch targets (buttons, install prompt CTA, offline banner dismiss): minimum 44px height — WCAG 2.5.5 compliance for kids using touchscreens.
- `--btn-height: 72px` — existing large button height for primary game CTAs, preserved as-is (source: `client/src/index.css:81`).
- PWA install prompt bottom offset: 24px from viewport bottom edge (`position: fixed; bottom: 24px`) — matches pattern in RESEARCH.md.
- Offline banner: 0px offset from top, full-width, 8px vertical padding only.

## Color

Ocean palette — all tokens already defined in `client/src/index.css:11–101`.

| Role | Value | Token | Usage |
|------|-------|-------|-------|
| Dominant (60%) | `#3BBFE8` | `--bg-primary` | Full-page ocean gradient background (body element), game canvas background |
| Secondary (30%) | `rgba(255,255,255,0.38)` | `--glass-bg` | Glass panels, cards, modals, install prompt card surface |
| Accent (10%) | `#6C5CE7` | (inline, matches PWA `theme_color`) | PWA install prompt background only — no other new UI in this phase uses this value |
| Destructive | `#EF4444` | `--accent-red` | Offline indicator banner background only |
| Error (semantic) | `#F87171` | `--error` | Error fallback screen text, toast error variant |

**Accent reserved for:** PWA install prompt background (`InstallPrompt.jsx`). No other new component in Phase 2 uses `#6C5CE7`. This matches the existing PWA `theme_color` in `vite.config.js` so the install prompt is visually consistent with the installed app chrome.

**Offline banner:** Uses `--accent-red` (`#EF4444`) not `--error` (`#F87171`) because the banner is a full-width status bar — it needs maximum contrast against the ocean background.

**Error fallback background:** Reuses the full ocean gradient (`linear-gradient(180deg, #8BD4F2 0%, #3BBFE8 60%, #E8C87A 88%, #D4A84B 100%)`) so a crash screen does not look alien in the kids UI.

**Toast colors:** Delegated to sonner's `richColors` prop — `toast.success` uses sonner's green, `toast.error` uses sonner's red, `toast()` (neutral) uses sonner's default. Do not override sonner's built-in color palette; it is accessible by default.

## Interaction States

### Toast Notifications (POL-01)

| Event | Toast Type | Copy | Duration |
|-------|-----------|------|----------|
| Coin reward | `toast.success` | `+{N} coins earned!` | 3000ms |
| Streak hit | `toast` (neutral with 🔥 icon) | `{N} day streak!` | 3000ms |
| Offline sync failure | `toast.error` | `Progress not saved — reconnect and try again` | 5000ms |
| Error (generic) | `toast.error` | `Something went wrong. Please try again.` | 4000ms |

**Position:** `top-center` (set on `<Toaster />` in `main.jsx`).
**richColors:** `true` — uses sonner's built-in accessible colors.
**Reduced-motion:** sonner respects `prefers-reduced-motion` natively; no override needed. The project already has a `@media (prefers-reduced-motion: reduce)` block in `index.css` — sonner will be compatible.

### Error Boundary (POL-02)

| State | What the user sees |
|-------|--------------------|
| Crash | Ocean gradient full-screen, 🐙 emoji at 64px, "Oops! Something went wrong" heading (28px Fredoka 600), "Try again" button (existing `.btn-primary` style or equivalent) |
| Reset | `window.location.assign('/')` — navigates to root, clears React tree |

### PWA Install Prompt (POL-03)

| State | What the user sees |
|-------|--------------------|
| First visit | Nothing — prompt suppressed |
| Second+ visit (Android Chrome) | Fixed pill at bottom: "Add KidsLearn to your home screen!" + "Install" button + ✕ dismiss |
| Second+ visit (iOS Safari) | Fixed pill: "Tap Share then 'Add to Home Screen'" + ✕ dismiss (no programmatic install API) |
| Already dismissed | Nothing — `localStorage.kl_pwa_dismissed` flag prevents repeat |
| After install | Nothing — browser fires `appinstalled` event; prompt self-hides |

### Offline Indicator (POL-04)

| State | What the user sees |
|-------|--------------------|
| Online | Banner not rendered (returns null) |
| Offline | Full-width red banner at top: "You are offline — progress may not save" |
| Reconnect | Banner immediately disappears (driven by `online` window event) |

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA — Error recovery | "Try again" |
| Primary CTA — PWA install (Android) | "Install" |
| PWA dismiss | "✕" (icon-only, aria-label="Dismiss") |
| PWA install body | "Add KidsLearn to your home screen!" |
| PWA install body (iOS) | "Tap Share then 'Add to Home Screen'" |
| Offline banner | "You are offline — progress may not save" |
| Error fallback heading | "Oops! Something went wrong" |
| Error fallback subtext | "Don't worry — Ollie will help you get back." |
| Toast — coin reward | "+{N} coins earned!" |
| Toast — streak | "{N} day streak!" |
| Toast — offline sync failure | "Progress not saved — reconnect and try again" |
| Toast — generic error | "Something went wrong. Please try again." |
| Empty state (no avatars) | Not applicable — Phase 2 does not introduce empty states; POL-07 is a constant extraction, not a new UI surface |
| Destructive confirmation | None — Phase 2 has no destructive user actions |

**Tone:** Simple, warm, encouraging. Fredoka font is inherently playful — copy follows suit. Error copy never uses technical language. Avoid words like "failed", "error code", "exception". Prefer "went wrong" over "failed", "reconnect" over "restore connection".

## Accessibility Notes

- All new fixed-position overlays (offline banner, install prompt) must have `role="status"` or `role="alert"` as appropriate, and must not trap keyboard focus (they are dismissible with no modal lock).
- `OfflineBanner` — use `role="alert"` so screen readers announce connectivity change immediately.
- `InstallPrompt` — use `role="complementary"` or `aria-label="Install app prompt"`. The ✕ dismiss button must have `aria-label="Dismiss install prompt"`.
- `ErrorFallback` — "Try again" button must be focusable and reachable without mouse. Entire error screen should have `role="main"`.
- Touch targets: all buttons in new components must be at minimum 44px in height (44px × 44px tap target). The existing `--btn-height: 72px` already satisfies this.
- Reduced-motion: no new CSS animations are introduced in Phase 2. Sonner respects `prefers-reduced-motion` natively.

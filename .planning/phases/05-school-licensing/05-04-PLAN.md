Structure: 4 sections -- License Status Card, Teachers List, Add Teacher Form, Invoices Table.

```jsx
import { useState, useEffect } from 'react';
import { api } from '../lib/api';
```

- `status` (null until loaded) -- from GET /api/billing/school-status
- `teachers` (array) -- from GET /api/school/teachers
- `invoices` (array) -- from GET /api/billing/school-invoices
- `newEmail` (string) -- input for adding teacher
- `loading` (boolean)
- `error` (string | null)

useEffect on mount: fetch all three endpoints in parallel using Promise.all:
```javascript
const [statusData, teacherData, invoiceData] = await Promise.all([
  api.get('/api/billing/school-status'),
  api.get('/api/school/teachers'),
  api.get('/api/billing/school-invoices'),
]);
```

If any fetch returns 403 (not a school admin), set error state "You are not a school admin" and render that message.

**Section 1: License Status Card**
- Display: school name, license status badge (active=green, expired=red, none=gray), expiry date formatted, seats used / seats total as "X / Y seats"
- If not licensed: show "Upgrade" buttons for tier_1, tier_2, tier_3 that POST to /api/billing/school-checkout and redirect to `data.url`
- If licensed: show "Manage Billing" link (for now, just the status -- portal can be added later)

**Section 2: Teachers List**
- Table/list of teachers with: name, email, role badge, classrooms count, "Remove" button (red)
- Remove button calls DELETE /api/school/teachers/:userId then refreshes teacher list
- Admin row has no remove button (per server validation)

**Section 3: Add Teacher Form**
- Input field for email + "Add Teacher" button
- POST /api/school/teachers with { email }
- On success: refresh teacher list, clear input
- On error: show error message (e.g., "No teacher account found", "Seat limit reached", "Already in a school")

**Section 4: Invoices Table**
- Table with: date (formatted from `created`), amount (amountPaid/100 with currency), status badge, "Download" link opening invoicePdf in new tab
- If no invoices: "No invoices yet" message

**Styling:** Use inline `styles` object pattern matching ParentDashboard.jsx. Use CSS variables from index.css: `var(--primary)`, `var(--surface)`, `var(--radius-lg)`, `var(--space-lg)`, `var(--font-lg)`, etc. Ocean theme colors.
